Le Projet

Le problème

Côté consommateur

  • De nombreux magasins ne proposent pas de scannettes ;
  • Indisponibilité/manque des scannettes ;
  • Problèmes techniques : batterie, UX, etc.

Côté super-marché

  • Coût d'installation et de maintenance du système ;
  • Gestion des scannettes (retour à l'entrée).

La solution

Côté consommateur 

  • Une application smartphone ;
  • Disponibilité et batterie à la charge de l'utiisateur ;
  • Possibilités techniques élargies :
    • Interface utilisateur, UX ;
    • Quantification de soi ;
    • Partage et échanges sociaux.

Côté super-marché

  • Infrastructure informatique ;
  • Couverture réseau sans fil ;
  • Base de données des produits ;
  • Encaissement.

Le Modèle Économique

Les clients

  • Principalement les super/hyper-marché (Auchan, Carrefour, Leclerc, Casino) ;
  • Les commerces de proximité (Carrefour City, Petit Casino, etc.) ;
  • Les grandes surfaces spécialisées (Castorama, Ikea, etc.).

Les revenus

Les 3 tiers des éditeurs logiciels :

  • Mise en place du service ;
  • Maintenance et mise à jour ;
  • Service client.

Possibilité de monétiser (dans une moindre mesure) l'application smartphone par la publicité.
